Chapter 6 – Analysis

Why Scale Estimation Exists

Architects estimate scale before proposing an architecture because architectural decisions should be supported by evidence rather than assumptions.

Key Mental Models

  • Requirements → Scale Estimation → Architecture
  • Architects estimate constraints, not numbers.
  • Every estimated metric should answer an architectural question.
  • Geographic distribution first affects latency; latency then influences deployment strategy.

Important Distinctions

  • Global users do not automatically imply more servers.
  • Performance and resilience are separate reasons for multi-region deployments.
  • Differentiate raw inputs (MAU, request size) from derived metrics (QPS, storage growth, throughput).
