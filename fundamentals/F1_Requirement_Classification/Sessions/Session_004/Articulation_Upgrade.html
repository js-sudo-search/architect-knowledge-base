Problem 4

Why does global geography influence architecture?

Original Answer

Global users affect server count, availability, redundancy and durability.

Communication Diagnosis
  • Correct direction.
  • Skipped the causal chain.
  • Server count was treated as the first consequence instead of latency.
Architect Articulation

Geographic distribution primarily increases network latency because requests travel longer distances. That drives regional deployments and intelligent request routing. Multi-region deployment can also improve resilience by allowing traffic to fail over if an entire region becomes unavailable.

Interview Takeaway

Always explain why geography changes the architecture before discussing technologies.

Problem 5

Which metrics would you estimate first?

Original Answer

Active users per geography, peak traffic, request size, bandwidth and availability.

Communication Diagnosis
  • Mixed raw inputs with derived metrics.
  • Availability is an NFR rather than a scale metric.
Architect Articulation

I would estimate read/write QPS, storage growth, throughput/bandwidth requirements and peak versus average load. Each estimate answers a specific architectural question about scaling, storage capacity, networking and capacity planning.

Interview Takeaway

For every metric, explain the architectural decision it enables.
