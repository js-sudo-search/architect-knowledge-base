Session 002 Insights

Architecture Principles

  • Functional Requirements describe capabilities.
  • Capabilities are identified before implementation choices.
  • Prioritize Functional Requirements before deriving NFRs.
  • Architects justify priorities using explicit criteria.

Interview Insights

  • Do not jump to Email/SMS or technology during requirement gathering.
  • Differentiate capabilities from UI specifications.
  • Explain prioritization criteria before listing priorities.

Common Mistakes

  • Confusing features with capabilities.
  • Confusing UI requirements with Functional Requirements.
  • Ranking requirements without explaining the decision framework.

Learning Capture

The biggest improvement was the transition from feature thinking to capability thinking. Functional Requirements are now viewed as a bridge between Business Objectives and Non-Functional Requirements.
