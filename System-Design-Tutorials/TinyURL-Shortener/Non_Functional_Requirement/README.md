# Non-Functional Requirements (NFR) – Architect's Quality Attributes Framework

Purpose
-------
Non-Functional Requirements define HOW WELL the system should perform its Functional Requirements.

Framework
---------
Business Objective
        ↓
Functional Requirements (What?)
        ↓
Non-Functional Requirements (How well?)
        ↓
Capacity Estimation (How much?)
        ↓
Architecture (How?)

Key Principle
-------------
Describe the quality expectation, not the implementation.
