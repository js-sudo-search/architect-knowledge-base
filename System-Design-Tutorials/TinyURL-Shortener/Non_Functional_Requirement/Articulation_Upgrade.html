Interview Problem

Describe the Non-Functional Requirements for a globally distributed URL Shortener.

Your Original Answer

You discussed storage, latency, peak traffic, availability and durability.

Strengths

  • Recognized the need for low latency and availability.
  • Connected durability to preserving URL mappings.

Gaps

  • Storage is an implementation concern, not an NFR.
  • Peak traffic and bandwidth belong to capacity estimation.
  • Availability should be framed around failures rather than traffic growth.

Refined Architect Answer

The key Non-Functional Requirements are: low latency for global redirects, high availability despite failures, scalability for future traffic growth, durability so URL mappings are never lost, and reliability so every short URL consistently resolves to the correct destination.

Why This Version Is Stronger

It describes quality expectations without prematurely discussing implementation.
