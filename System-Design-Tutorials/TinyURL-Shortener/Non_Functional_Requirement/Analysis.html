Analysis – Non-Functional Requirements

Definition

Functional Requirements define what the system should do. Non-Functional Requirements define the quality attributes that describe how well it should perform those functions.

Quality Attributes for URL Shortener

NFRWhy it matters
Low LatencyUsers expect redirects to complete quickly regardless of location.
High AvailabilityLinks should continue working despite infrastructure failures.
ScalabilityThe system should continue to perform as traffic grows.
DurabilityURL mappings should never be lost once created.
ReliabilityUsers should consistently be redirected to the correct destination.

Common Mistakes

  • Mixing storage, databases, caches, or load balancers into NFRs.
  • Confusing capacity planning (bandwidth, QPS, headroom) with quality attributes.
  • Describing implementation instead of expectations.

Mental Model

FR  : What should the system do?
NFR : How well should it do it?
Capacity : How much should it handle?
Architecture : How will we build it?
